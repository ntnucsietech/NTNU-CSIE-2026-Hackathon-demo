## 在開始之前

本指南說明如何修改遊戲中的角色數值、商店內容與音訊設定。
所有修改都只需要用文字編輯器（如 VS Code）開啟對應檔案，找到要改的地方直接修改後存檔即可。

> ⚠️ 修改前建議先備份原始檔案，避免改錯後無法還原。

## 角色數值與資源

所有數值都定義在 **`new_data.js`** 裡。

---

### 玩家初始數值

搜尋 `playerStats`，找到這段：

```javascript
// new_data.js
var playerStats = {
  name:   "勇者",
  hp:     1000,     // 初始 HP
  maxHp:  1000,     // 最大 HP 上限
  atk:    10,       // 攻擊力
  def:    5,        // 防禦力
  spd:    20,       // 速度（影響命中率與爆擊率，不顯示在畫面上）
  money:  10000,    // 初始金幣
  keys:   0,        // 初始鑰匙數
  skills: ["power_strike"]  // 初始技能（填技能的 id）
};
```

直接修改對應數字即可。例如把初始金幣改成 `500`：

```javascript
money: 500,
```

```javascript

  reward: { money: 150 }
```

每個敵人物件的欄位說明：

| 欄位 | 說明 |
| ---- | ---- |
| `name` | 敵人名稱 |
| `tier` | 所在區域：`"A"`（前段）/ `"B"`（中段）/ `"C"`（後段） |
| `hp` / `maxHp` | 血量 / 最大血量 |
| `atk` | 攻擊力 |
| `def` | 防禦力 |
| `spd` | 速度（影響命中率與爆擊率） |
| `reward.money` | 擊敗後獲得的金幣 |
| `isPaired` | `true` 表示會以「雙體」形式出現 |
| `isMiniBarrier` | `true` 表示是小遊戲關卡的守門怪，行為特殊 |

### 寶箱獎勵

搜尋 `var chestRewards`，每個物件代表一種可能的寶箱內容（隨機抽取一種）：

```javascript
var chestRewards = [
  { money: 25,  message: "你找到了 25 枚金幣！" },
  { atk:   3,   message: "你找到了力量秘藥，攻擊力永久提升 3！" },
  { def:   2,   message: "你找到了盾牌碎片，防禦力永久提升 2！" },
  { money: 35,  message: "大寶箱！你找到了 35 枚金幣！" },
  { reviveAlly: true, message: "你找到了友軍復活藥水！" }
];
```

`message` 是打開寶箱時顯示的提示文字，可以直接修改。
`money`、`atk`、`def` 對應獎勵的數值，可以自由調整。

### 命中率與爆擊率

搜尋 `SPD_HIT_SCALE`：

```javascript
var SPD_HIT_SCALE    = 1.5;   // 速度差對命中率/爆擊率的影響比例（越小影響越大）
var BASE_ATTACK_HIT  = 75;    // 普通攻擊基礎命中率 %（調低 = 閃避率提高）
var BASE_CRIT_RATE   = 0.05;  // 爆擊率下限（任何角色至少有 5% 爆擊率）
var SKILL_CRIT_BONUS = 0.15;  // 使用技能時額外增加的爆擊率
```

* **命中率公式**：`基礎命中率 + (攻擊方速度 - 防禦方速度) / SPD_HIT_SCALE`，結果限制在 30% ～ 95%
* **爆擊率公式**：`(攻擊方速度 - 防禦方速度) / SPD_HIT_SCALE`，結果限制在 `BASE_CRIT_RATE` ～ 90%
* 使用技能時爆擊率額外加上 `SKILL_CRIT_BONUS`

### 若想移除隨身商店並添加多個不同商店

目前遊戲的商店設計是「全域隨身商店」：玩家第一次踩到商店格後解鎖，之後可隨時用 `F` 鍵或右側商店按鈕開啟，所有商店格都顯示同一份商品清單。

若想改成「地圖上有多個位置固定、各自賣不同商品的商店」，需要修改以下四個地方。

---

#### Step 1 — 移除隨身商店（F 鍵 / HUD 按鈕）

打開 **`new_engine.js`**，找到 `triggerShop` 函式，把 `shopUnlocked = true` 這行以及後續的 `updateControlsHint()` 呼叫移除或註解掉：

```javascript
// new_engine.js
function triggerShop() {
  // 移除以下兩行，讓 F 鍵 / HUD 按鈕永遠不解鎖
  // var isFirst = !shopUnlocked;
  // shopUnlocked = true;
  // updateControlsHint();

  openShop(shopItems);  // 改為傳入商品清單（見 Step 2）
}
```

接著找到鍵盤處理區，把 F 鍵開啟商店的那一行刪除或註解：

```javascript
// new_engine.js — 搜尋「key === "f"」
// if ((e.key === "f" || e.key === "F") && shopUnlocked) { openShop(); return; }
```

最後在 **`index.html`** 找到商店按鈕並隱藏或刪除：

```html
<!-- index.html — 找到並刪除或直接加上 style="display:none" -->
<button class="btn btn-side-action btn-side-shop" id="btn-open-shop"
        onclick="openShopFromHUD()" style="display:none">🛒 商店</button>
```

---

#### Step 2 — 為每間商店定義不同商品清單

打開 **`new_data.js`**，在 `shopItems` 下方新增你想要的商店商品組合。格式與 `shopItems` 完全相同：

```javascript
// new_data.js
// 武器商店（只賣攻擊相關）
var shopItems_weapon = [
  { name: "攻擊強化藥水", price: 25, effect: { atk: 3 }, desc: "攻擊力永久 +3", isConsumable: false },
  { name: "攻擊爆發劑",   price: 18, effect: { tempAtk: 12 },
    desc: "戰鬥中使用：指定對象本場 ATK +12", isConsumable: true, targetSide: "ally", targetType: "single" },
];

// 藥水商店（只賣回復相關）
var shopItems_potion = [
  { name: "血量恢復藥水", price: 10, effect: { hp: 30 },   desc: "立即回復 30 HP", isConsumable: false },
  { name: "大恢復藥水",   price: 25, effect: { hp: 80 },   desc: "立即回復 80 HP", isConsumable: false },
  { name: "治癒藥水",     price: 16, effect: { hp: 50 },
    desc: "戰鬥中使用：指定對象回復 50 HP", isConsumable: true, targetSide: "ally", targetType: "single" },
];
```

---

#### Step 3 — 修改 `openShop` 接受商品清單參數

打開 **`new_engine.js`**，找到 `function openShop()`，在函式宣告加上一個參數，讓它可以接收不同商品清單：

```javascript
// new_engine.js — 修改 openShop 函式宣告
function openShop(itemList) {
  itemList = itemList || shopItems;  // 預設使用全域 shopItems

  var list = document.getElementById("shop-item-list");
  if (!list) return;
  list.innerHTML = "";

  // 把原本寫死的 shopItems 全部換成 itemList
  sec("🔧 永久道具");
  itemList.filter(function(it) { return !it.isConsumable; })
          .forEach(function(it) { renderShopCard(it, list); });

  sec("🧪 戰鬥消耗品");
  itemList.filter(function(it) { return it.isConsumable; })
          .forEach(function(it) { renderShopCard(it, list); });

  // 技能 / 合成 / 同伴區塊維持不變，不需修改
  // ...
```

> ⚠️ 只需修改第一行（加參數）和篩選商品的兩個 `shopItems` → `itemList`，其餘技能、同伴區塊不動。

---

#### Step 4 — 在地圖上指定哪個格子開哪間商店

打開 **`map.js`**，利用現有的 `tileEnemyMap`（指定格子對應特定怪物）概念，仿照新增一個 `tileShopMap`，指定每個商店格要開哪份商品清單：

```javascript
// map.js — 加在 tileEnemyMap 附近
var tileShopMap = {
  "5,6":  "weapon",   // 第 6 列第 5 欄的商店格 → 武器商店
  "11,9": "potion",   // 第 9 列第 11 欄的商店格 → 藥水商店
  // 格式：「x,y」: 「商店名稱」
};
```

> 座標格式為 `"x,y"`，可用瀏覽器開發工具或地圖編輯器查看格子位置。

接著回到 **`new_engine.js`**，修改 `triggerShop` 讓它讀取 `tileShopMap` 來決定開哪間商店：

```javascript
// new_engine.js
function triggerShop(x, y) {
  var key      = x + "," + y;
  var shopName = (typeof tileShopMap !== "undefined" && tileShopMap[key]) || "default";

  var itemList;
  if      (shopName === "weapon") itemList = shopItems_weapon;
  else if (shopName === "potion") itemList = shopItems_potion;
  else                            itemList = shopItems;  // 預設商店

  openShop(itemList);
}
```

同時要讓 `triggerShop` 被呼叫時帶入座標。找到 `checkTileEvent`，把原本的 `triggerShop()` 改成 `triggerShop(x, y)`：

```javascript
// new_engine.js — 搜尋 checkTileEvent
else if (t === MAP_TILE.SHOP) triggerShop(x, y);  // 加上 x, y 參數
```

---

#### 完成後的效果

* 地圖上的商店格各自獨立，賣不同商品
* F 鍵和右側 HUD 按鈕已移除，玩家只能走到商店格才能購物
* 新增商店只需：① 在 `new_data.js` 定義新的商品陣列 → ② 在 `tileShopMap` 加上對應座標 → ③ 在 `triggerShop` 加上對應的 `else if`

## 音訊

所有音訊設定都在 **`audio.js`** 裡，分成三個區塊。

> 音檔統一放在 `assets/audio/` 資料夾下。支援 `.mp3` 和 `.wav` 格式。
> 瀏覽器限制：音訊需要使用者**先點擊畫面一次**後才會開始播放，這是正常現象。

---

### 音效（SFX）

搜尋 `var SFX_CONFIG`，每個 key 對應一個遊戲事件的音效：

```javascript
var SFX_CONFIG = {
  encounter  : null,   // 進入戰鬥（尚無音檔）
  phy_attack : null,   // 物理攻擊（預留）
  mag_attack : null,   // 魔法攻擊（預留）
  shield     : null,   // 防禦（預留）
  dodge      : { src: "assets/audio/SFX/dodge.mp3",  volume: 0.5 },  // 閃避
  attack     : { src: "assets/audio/SFX/attack.wav", volume: 1.0 },  // 攻擊命中
  flash_token: { src: "assets/audio/SFX/flash.wav",  volume: 0.8 },  // ◆ 轉 ◈
  victory    : null,   // 擊敗敵人
  defeat     : null,   // 玩家死亡
  flee       : null,   // 逃跑成功
  chest      : { src: "assets/audio/SFX/open_chest.mp3", volume: 0.5 },  // 打開寶箱
  locked_door: null,   // 嘗試開鎖住的門
  unlock_door: null,   // 開門成功
  teleport   : { src: "assets/audio/SFX/teleport.mp3",   volume: 1.0 },  // 傳送門
  buy        : { src: "assets/audio/SFX/buy.wav",        volume: 1.0 },  // 購買商品
  shot       : { src: "assets/audio/SFX/shot.wav",       volume: 1.0 },  // 射擊小遊戲擊中
  key        : null,   // 小遊戲通關得鑰匙
};
```

填入音效的方式有兩種：

**方式一**：只填路徑（音量預設為最大）
```javascript
flee: "assets/audio/SFX/flee.mp3",
```

**方式二**：填路徑加自訂音量（`0.0` = 靜音，`1.0` = 最大）
```javascript
flee: { src: "assets/audio/SFX/flee.mp3", volume: 0.7 },
```

暫時不想填音效的項目，填 `null` 保留空位即可，不會影響遊戲運作。

### 各種攻擊/技能聲音

遊戲中的攻擊與戰鬥音效都定義在 **`audio.js`** 的 `SFX_CONFIG` 裡。以下是目前所有戰鬥相關的音效 key 與觸發時機：

| key | 觸發時機 | 目前狀態 |
| --- | -------- | -------- |
| `attack` | 任何一方發動攻擊命中時 | `attack.wav` |
| `dodge` | 任何一方攻擊 Miss 時 | `dodge.mp3` |
| `flash_token` | 任何一方爆擊，◆ 轉為 ◈ 時 | `flash.wav` |
| `phy_attack` | 預留：物理攻擊音效 | 尚無音檔 |
| `mag_attack` | 預留：魔法攻擊音效 | 尚無音檔 |
| `shield` | 預留：防禦/格擋音效 | 尚無音檔 |

搜尋 `var SFX_CONFIG`，找到對應的 key 填入音檔路徑即可：

```javascript
// audio.js — SFX_CONFIG 戰鬥相關區塊
attack     : { src: "assets/audio/SFX/attack.wav", volume: 1.0 },
dodge      : { src: "assets/audio/SFX/dodge.mp3",  volume: 0.5 },
flash_token: { src: "assets/audio/SFX/flash.wav",  volume: 0.8 },
phy_attack : null,   // 填入路徑後即可啟用
mag_attack : null,
shield     : null,
```

填入的方式和其他音效相同，支援純字串或含音量的物件格式：

```javascript
// 只填路徑（音量預設 1.0）
phy_attack: "assets/audio/SFX/sword.wav",

// 填路徑加自訂音量
phy_attack: { src: "assets/audio/SFX/sword.wav", volume: 0.8 },
```

> `phy_attack`、`mag_attack`、`shield` 為預留 key，填入路徑後需自行在 `new_engine.js` 找到對應的攻擊邏輯，加上 `playSound("phy_attack")` 等呼叫才會有聲音。

### 背景音樂（BGM）

搜尋 `var BGM_TRACKS`，每個 key 是一首 BGM 音軌的設定：

```javascript
var BGM_TRACKS = {
  map: {
    src       : null,                        // 地圖 BGM（尚無音檔）
    volume    : 0.6,
    loopStart : 0,
    loopEnd   : null
  },
  battle1: {
    src       : "assets/audio/bgm_battle1.mp3",   // 一般戰鬥 BGM
    volume    : 0.8,
    loopStart : 0,
    loopEnd   : null
  },
  boss: {
    src       : "assets/audio/bgm_boss.mp3",      // Boss 戰 BGM
    volume    : 0.9,
    loopStart : 0,
    loopEnd   : null
  },
  minigame: {
    src       : "assets/audio/mini_game.mp3",     // 射擊小遊戲 BGM
    volume    : 0.9,
    loopStart : 0,
    loopEnd   : null
  },
  // 其他音軌（shop / battle2 / clear / gameover）目前 src 為 null，可自行填入
};
```

欄位說明：

| 欄位 | 說明 |
| ---- | ---- |
| `src` | 音檔路徑，填 `null` 表示此情境靜音 |
| `volume` | 這首曲子的基礎音量（`0.0` ～ `1.0`） |
| `loopStart` | 循環起始點（秒）。`0` 表示從頭循環 |
| `loopEnd` | 循環結束點（秒）。填 `null` 表示播到結尾後循環 |

**Loop 設定範例**：有前奏、只循環中間段

```javascript
battle1: {
  src       : "assets/audio/bgm_battle1.mp3",
  volume    : 0.8,
  loopStart : 4.2,    // 播到 58.3 秒時，跳回第 4.2 秒繼續
  loopEnd   : 58.3    // 開頭 0 ～ 4.2 秒的前奏只播一次
},
```

### 情境對應表

搜尋 `var SITUATION_BGM`，設定每個遊戲畫面對應播哪首 BGM：

```javascript
var SITUATION_BGM = {
  map      : "map",       // 地圖畫面 → 播 BGM_TRACKS.map
  combat   : "battle1",   // 一般戰鬥 → 播 BGM_TRACKS.battle1
  boss     : "boss",      // Boss 戰   → 播 BGM_TRACKS.boss
  shop     : "shop",      // 商店畫面 → 播 BGM_TRACKS.shop
  minigame : "minigame",  // 射擊小遊戲 → 播 BGM_TRACKS.minigame
  gameover : "gameover",  // 遊戲結束 → 播 BGM_TRACKS.gameover
  clear    : "clear",     // 通關畫面 → 播 BGM_TRACKS.clear
  dialogue : null         // 對話框   → 不切換，維持當前 BGM
};
```

右邊填的是 `BGM_TRACKS` 裡的 key 名稱。想讓某個畫面靜音，把值改成 `null` 即可：

```javascript
shop: null,   // 進入商店時靜音
```

> Boss 戰的 BGM 切換是由程式自動判斷，不需要額外設定。
