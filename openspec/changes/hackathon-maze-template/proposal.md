# Proposal: Hackathon Maze RPG Template

## Context
為資工營黑客松製作一個「迷宮 RPG」的網頁模板，提供給參賽學員使用。因為參賽者包含沒有太多網頁開發經驗的新手，模板必須非常容易理解和修改。

系統需具備以下主要功能：
- 前端網頁遊玩（無後端強制需求）。
- 固定網格迷宮（預設 10×10），霧中探索視角（玩家只能看到附近一定範圍內的格子，其餘遮蔽）。
- 玩家角色以 Sprite 圖片呈現，預設由 AI 生成，學員可自行替換。
- 地圖事件格（不同事件顯示不同外觀）：
  - 小 Boss / 一般敵人：觸發戰鬥畫面
  - 寶箱：獲得物品（金幣、血量、攻擊力強化）
  - 鑰匙房間（小遊戲格）：完成射擊小遊戲以獲得鑰匙
  - 門：需要持有鑰匙才能解鎖
  - 商店格：首次踩到後永久可用，學員亦可設定為出生點旁的格子（不佔用主要地圖網格）
  - 最終 Boss 格：擊敗後遊戲通關
- 回合制戰鬥系統，戰鬥介面預先設計完成，學員決定攻擊、防禦等邏輯細節。
- 商店機制：花費金幣購買能力加成。

學員可自定義的實作（依能力分層）：
- **必要**：填入角色/敵人數值、對話劇情文本。
- **核心**：選擇一種戰鬥模式（傳統 RPG 或 Press Turn），使用提供的戰鬥 Hook 撰寫攻擊、防禦等回合邏輯。
- **進階**：修改地圖陣列、隨機迷宮生成、角色 Sprite 自行繪製替換、音效素材替換。

限制：
- HTML 語法需維持最基礎的標籤（`div`, `h1`, `button`, `img`, `p` 等）。
- CSS 允許使用基礎的 Flexbox 版面配置（最多到 Flex），但絕不使用 CSS Grid，並可搭配基本的過場與放大效果（例如 `:hover`, `transition`, `scale`）。
- JS 允許使用 `const`/`let` 與 `forEach`，但禁止使用 `class`、`Promise`、箭頭函式（`=>`）。允許全域變數管理狀態，但所有狀態更新須透過明確命名的函式進行。
- 音效功能（預設音效素材）列為「待補充」，本版本不實作，架構預留位置即可。

## Proposed Change
設計一套**職責分離**的框架架構：
1. **Engine Core (不可修改區)**: 負責遊戲循環管理、網格繪製（含霧中探索遮罩）、鍵盤移動監聽與地圖碰撞偵測、狀態更新函式庫。
2. **Data Config (可修改核心)**: 學員透過單純的 JavaScript 物件 (Object) 與二維陣列 (2D Array) 來定義地圖結構、敵人數值、對話文本與物品。
3. **Combat Module (可修改邏輯)**: 戰鬥介面 UI 由框架預先設計，提供 Hook 讓學員撰寫攻擊/防禦的回合結算邏輯（傳統 RPG 模式為範例）。
4. **Mini-game Module**: 提供一個預設的射擊小遊戲（拖動滑鼠移動準心以射擊敵人），過關後返回布林值交由引擎判斷是否發放獎勵。

## Alternatives Considered
- 使用現成的 Canvas/WebGL 遊戲引擎 (如 Phaser)：對於初學者來說太難以理解其渲染邏輯，且不符合「只使用最基礎 HTML/CSS」的要求。
- 完全不給任何核心框架：黑客松時間有限，且很多學員無法從零建出網格移動系統，這將導致大家都在花時間在移動邏輯而非遊戲特色。
- 即時制戰鬥：對新手而言互動時序控制更複雜，回合制可以完全使用簡單的函式呼叫順序來實作。
